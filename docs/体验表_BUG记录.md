## 详细描述

### BUG-001：知识库内容为空

- **问题**：知识库里没有内容。
- **预期**：知识库应包含相关的知识数据，供系统和 Agent 读取使用。
- **状态**：待修复

---

### BUG-002：老板决策界面重复

- **问题**：老板决策功能目前存在两个界面入口——一个是弹窗形式，另一个是右下角的浮动窗口，体验混乱。
- **预期**：统一为一个界面入口，保持交互一致性。
- **状态**：待修复

---

### BUG-003：Agent 未产出文档且未读取知识库

- **问题**：Agent 依然没有产出对应的文档内容，同时也没有统一读取知识库里的内容。
- **预期**：Agent 应能正确读取知识库内容，并根据流程产出对应的文档。
- **状态**：待修复

---

### BUG-004：决策界面信息量不足，无法做出有效判断

- **问题**：决策功能虽然存在，但展示的信息非常简单，用户无法清晰地看到需要决策的具体内容。例如一开始的游戏方案，只有一个简单的介绍就要求用户进行决策，缺少充分的背景信息和细节支撑，导致用户不知道该决策什么。
- **预期**：决策界面应提供充足的信息量，包括但不限于：
  - 方案的详细说明（背景、目标、核心玩法等）
  - 关键数据或对比分析（如有多个方案应列出各自优劣）
  - 涉及的资源与风险评估
  - 明确的决策选项及其影响说明
- **核心诉求**：用户需要足够多的信息才能做出准确判断，不能只靠一句话概括就要求拍板。
- **状态**：待修复

---

### BUG-005：Agent 列表中"文件界面"功能用途不明

- **问题**：在 Agent 列表里看到一个"文件界面"，但用户完全不清楚这个界面是干什么用的，缺少功能说明或引导。
- **预期**：每个功能模块应有清晰的用途说明或引导提示，让用户一眼就能理解该功能的作用。
- **状态**：待确认

---

### BUG-006：功能状态显示"开发中"与实际进度不符

- **问题**：Agent 列表中部分功能显示为"开发中"，但项目实际上已经开发到 P10 阶段。用户困惑于：这里的"开发中"指的是什么？P10 阶段对应的具体内容是什么？功能状态与实际开发进度之间的关系不清晰。
- **预期**：
  - 功能状态应与实际开发进度保持一致，及时更新
  - 如果某些功能确实还在开发中，应说明预计完成的阶段
  - 用户应能清楚了解当前所处阶段（如 P10）对应的功能范围
- **状态**：待确认

---

### BUG-007：整体界面设计交互和布局不够灵活合理，需重新设计

- **问题**：目前界面的交互设计和布局不够灵活、不够合理，整体视觉和使用体验与现代 Web 应用有差距。
- **预期**：
  - 界面布局应更加灵活、合理，符合现代 Web 设计规范
  - 交互逻辑应清晰流畅，减少用户困惑
  - 视觉风格应现代、专业、简洁
- **改进建议**：在修改 UI 界面时，应参考 Vercel 出品的 v0（v0.dev）等现代 UI 设计工具的能力和设计理念，借助其生成的高质量 UI 组件和布局方案来重新设计界面。
- **参考资源**：
  - [v0.dev](https://v0.dev) — Vercel 出品的 AI UI 生成工具，可用于快速生成现代化的界面设计方案
  - 关注 shadcn/ui、Tailwind CSS 等现代前端 UI 体系的设计规范
- **状态**：待修复

---

  - 这个区域的功能定位是什么？是用来展示已完成的游戏成品吗？

  - 左下角的像素小人头像和表情图标有什么含义？

### BUG-009：Agent 产出的 .md 文档无法打开查看

- **问题**：Agent 产出了 .md 格式的文档，但用户点击后无法打开查看内容。
- **预期**：用户应能直接点击打开并查看 Agent 生成的 .md 文档，支持 Markdown 渲染预览。
- **状态**：待修复

---

### BUG-010：Agent 产出的文档未同步到知识库，知识库始终为空

- **问题**：Agent 产出了文档，但这些文档没有被自动同步到知识库中，导致知识库始终是空的。这与 BUG-001（知识库为空）和 BUG-003（Agent 未读取知识库）形成关联——整条链路断裂：Agent 不写入知识库 → 知识库为空 → Agent 无法读取知识库内容。
- **预期**：
  - Agent 产出的文档应自动同步写入到知识库
  - 知识库应实时反映最新的文档产出
  - 其他 Agent 应能从知识库中读取到这些文档作为上下文参考
- **关联问题**：BUG-001、BUG-003
- **状态**：待修复

---

### BUG-011：Agent 需要增加进度条和"上帝视角"实时工作画面

- **问题**：目前每个 Agent 只显示一个简单的"工作中"状态，用户完全看不到 Agent 实际在做什么、做到哪里了、产出了什么。缺乏透明度，无法判断 Agent 是否在正确地工作。
- **预期（上帝视角）**：用户应能像看到员工电脑桌面一样，实时看到每个 Agent 的具体工作内容，例如：
  - **程序员 Agent**：能看到正在编写的代码、当前在处理哪个模块
  - **策划 Agent**：能看到正在撰写的文档内容、方案进展
  - **美术 Agent**：能看到正在生成的图片、设计稿产出
- **预期（进度条）**：每个 Agent 应有明确的进度条或阶段指示，展示：
  - 当前任务总共有几个步骤
  - 已完成哪些步骤、当前在哪一步
  - 预估剩余时间或完成百分比
- **核心诉求**：用户作为"老板"需要对每个 Agent 的工作进度和内容有全面的掌控力，而不是只看到一个空洞的"工作中"。
- **状态**：待修复

---

### BUG-012：Agent 长时间卡在"工作中"，无法排查原因

- **问题**：程序员 Agent 显示"工作中"后，等了很长时间都没有结束，也无法查看是什么原因导致卡住。用户没有任何手段来诊断问题。
- **预期**：
  - 每个 Agent 应有超时检测机制，超过预期时间应主动提示异常
  - 提供日志或状态详情入口，用户可以查看 Agent 当前卡在哪个环节
  - 提供"重试"或"中断并重启"按钮，让用户可以干预卡住的 Agent
  - 如果是因为依赖其他 Agent 的产出而等待，应明确告知等待原因
- **关联问题**：BUG-011（如果有上帝视角就能直观看到卡住的原因）
- **状态**：待修复
