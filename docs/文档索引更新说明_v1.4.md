## 🎯 新增内容

### 1. 文档存放位置规范

添加了详细的文档分类和存放规则，核心要点：

#### ✅ 适用范围

**本规范仅适用于开发"游戏开发平台"本身的过程**

这是关键区分点，确保 AI 开发者明确：
- 平台开发文档 ≠ 游戏项目文档
- 规范只约束平台开发，不影响平台运行

#### 📂 明确区分两类文档

| 文档类型 | 存放位置 | 示例 |
|---------|---------|------|
| **平台开发文档** | `docs/` 目录 | 开发计划、测试报告、阶段总结 |
| **游戏项目文档** | `projects/{项目名}/` | GDD、TDD、共享知识库 |

#### 🚫 禁止行为

明确列出了三个禁止事项：
1. ❌ 禁止在根目录创建开发文档（除 README.md）
2. ❌ 禁止在代码目录下创建文档
3. ❌ 禁止创建重复文档

#### 📝 强制规则表格

提供了清晰的文档类型与存放位置对照表：

```
阶段完成报告 → docs/P5_阶段完成报告.md
测试报告     → docs/P5_测试报告.md
技术文档     → docs/WebSocket实现说明.md
改进措施     → docs/性能优化方案.md
总结报告     → docs/里程碑总结.md
```

#### 💡 AI 开发者注意事项

添加了 4 条关键提醒：
1. 阶段完成后，报告必须放在 `docs/`
2. 新增文档后，必须更新索引
3. 遵循命名规范
4. 不要混淆两类文档

## 🎯 为什么需要这个规范？

### 问题背景

在之前的开发中发现：
- 部分文档创建在根目录，导致混乱
- 文档位置不统一，查找困难
- AI 可能不清楚文档应该放在哪里

### 解决方案

通过明确规范：
1. ✅ 统一所有平台开发文档到 `docs/` 目录
2. ✅ 明确区分平台文档和游戏项目文档
3. ✅ 提供清晰的示例和禁止事项
4. ✅ 建立可执行的规则

## 📝 使用建议

### 对于 AI 开发者

**每次创建文档前，请检查**：
1. 这是平台开发文档还是游戏项目文档？
   - 平台开发 → `docs/` 目录
   - 游戏项目 → `projects/{项目名}/` 目录

2. 文档名称是否符合规范？
   - 清晰、有意义
   - 不过长、不过短
   - 使用标准格式

3. 是否需要更新文档索引？
   - 新增文档 → 必须更新
   - 修改现有文档 → 更新修改记录

### 对于用户

**查找文档时**：
- 平台开发相关 → 查看 `docs/` 目录
- 游戏项目相关 → 查看 `projects/` 目录
- 不确定 → 查看 `docs/文档索引.md`
