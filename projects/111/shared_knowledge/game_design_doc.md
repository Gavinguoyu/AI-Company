## 游戏策划文档 (GDD)

### 1. 游戏名称
**《Sky Jumper》**

### 2. 游戏概述
《Sky Jumper》是一款操作简单、极具挑战性的休闲街机游戏。玩家将扮演一只可爱的小鸟，通过点击屏幕控制其飞行高度，在不断前进的管道森林中穿梭，目标是尽可能地获得高分，挑战自己的反应速度和操控技巧。

### 3. 核心玩法
*   **操作**: 玩家通过单次点击屏幕（或鼠标左键）使小鸟获得一个向上的推力，从而向上飞行一小段距离。
*   **重力**: 小鸟持续受到重力影响，在没有操作时会自然下落。
*   **穿梭**: 玩家需要精确控制小鸟的飞行高度，使其从上下两根管道形成的空隙中穿过。
*   **得分**: 每成功穿过一对管道，玩家获得1分。
*   **失败**: 小鸟与任何管道、地面或屏幕顶部边缘发生碰撞时，游戏结束。

### 4. 玩家角色
*   **名称**: 小鸟 (Bird)
*   **外观**: 可爱、卡通风格的小鸟形象，建议有简单的翅膀拍打动画。
*   **物理特性**:
    *   受重力影响，持续下落。
    *   每次点击获得一个瞬间向上的速度（跳跃力）。
    *   有最大下落速度限制，避免速度过快导致难以控制。
*   **碰撞体**: 定义为矩形或圆形（具体取决于美术资源和物理引擎实现，建议圆形或胶囊体以平滑碰撞）。

### 5. 障碍物
*   **类型**: 管道 (Pipes)
*   **构成**: 由一根上管道和一根下管道组成，中间留有固定高度的空隙。
*   **生成**:
    *   管道从屏幕右侧外部生成，以固定速度向左移动。
    *   每对管道之间的水平距离固定。
    *   每对管道的整体垂直位置（即空隙的中心点Y坐标）在一定范围内随机变化。
*   **销毁**: 当管道完全移出屏幕左侧时，进行销毁以优化性能。
*   **碰撞体**: 定义为矩形。

### 6. 计分系统
*   **得分规则**: 每当小鸟的X坐标成功越过一对管道的右边缘时，获得1分。
*   **显示**: 实时显示当前分数。
*   **最高分**: 记录并显示玩家的历史最高分数。

### 7. 游戏状态
*   **标题/开始界面 (Title/Start Screen)**:
    *   显示游戏标题。
    *   显示“开始游戏”按钮。
    *   显示历史最高分。
    *   可展示游戏玩法提示（如“点击屏幕跳跃”）。
*   **游戏进行中 (Playing State)**:
    *   小鸟、管道、背景持续运动。
    *   实时显示当前分数。
    *   玩家可进行操作。
*   **游戏结束界面 (Game Over Screen)**:
    *   显示“游戏结束”提示。
    *   显示本次游戏得分。
    *   显示历史最高分。
    *   提供“重新开始”按钮。
    *   提供“返回主菜单”按钮。

### 8. 用户界面 (UI)
*   **分数显示**: 游戏过程中，屏幕顶部中央显示当前分数。
*   **按钮**: 开始、重新开始、返回主菜单等按钮，需有清晰的视觉提示和点击反馈。

### 9. 音效与视觉
*   **音效**:
    *   **跳跃音效**: 每次点击小鸟上升时播放。
    *   **得分音效**: 成功穿过管道时播放。
    *   **碰撞音效**: 游戏结束时播放（如“砰”或“坠落”音）。
    *   **背景音乐**: 轻快活泼的循环音乐。
*   **视觉**:
    *   整体采用2D卡通风格。
    *   背景：循环滚动的云层或城市背景，营造前进感。
    *   小鸟动画：简单的拍打翅膀动画。
    *   管道：简洁明了的几何形状，颜色与背景有良好对比度。

### 10. 技术要求与注意事项
*   **物理系统**: 需要一个简单的2D物理系统来处理重力、跳跃力和碰撞。
*   **碰撞检测**: 采用高效的碰撞检测机制（如AABB或圆形碰撞），确保在移动物体数量多时仍能保持性能。
*   **资源管理**: 背景、管道等资源的循环复用，避免频繁创建销毁，提高性能。
*   **多平台适配**: 考虑不同屏幕分辨率和宽高比的UI和游戏区域适配。

## 游戏配置表 (YAML格式)

以下是《Sky Jumper》的核心数值配置表，程序员可以直接导入使用。所有数值均已添加详细注释，方便理解和调整。

```yaml
# game_config.yaml

# ---------------------------------------------------
# 游戏全局设置 (Game Global Settings)
# ---------------------------------------------------
game_settings:
  game_speed: 200             # 游戏整体水平移动速度，单位：像素/秒。此值越大，管道移动越快，游戏难度越高。
  gravity: 1000               # 角色受到的重力加速度，单位：像素/秒^2。此值越大，小鸟下落越快。
  floor_y_position: 100       # 地面Y坐标，单位：像素 (从屏幕底部向上计算)。小鸟Y坐标小于此值视为碰撞地面。
  ceiling_y_position: 0       # 天花板Y坐标，单位：像素 (从屏幕顶部向下计算)。小鸟Y坐标大于此值视为碰撞天花板。
  initial_screen_width: 1280  # 游戏设计时的标准屏幕宽度，用于计算相对位置。
  initial_screen_height: 720  # 游戏设计时的标准屏幕高度，用于计算相对位置。

# ---------------------------------------------------
# 玩家角色设置 (Player Character Settings)
# ---------------------------------------------------
player_settings:
  jump_force_velocity: 400    # 每次点击屏幕时，小鸟获得的垂直向上速度，单位：像素/秒。
  max_fall_speed: 500         # 小鸟的最大下落速度，单位：像素/秒。防止下落过快导致无法反应。
  collision_box_width: 40     # 玩家碰撞体的宽度，单位：像素。
  collision_box_height: 40    # 玩家碰撞体的高度，单位：像素。
  initial_position_x: 200     # 玩家初始X坐标 (相对于屏幕左侧)。
  initial_position_y: 360     # 玩家初始Y坐标 (相对于屏幕顶部)。

# ---------------------------------------------------
# 管道障碍物设置 (Pipe Obstacle Settings)
# ---------------------------------------------------
pipe_settings:
  pipe_width: 80              # 单根管道的宽度，单位：像素。
  pipe_gap_height: 200        # 上下管道之间的空隙高度，单位：像素。此值越小，难度越高。
  horizontal_spacing: 300     # 相邻两对管道之间的水平间距，单位：像素。此值越小，管道越密集，难度越高。
  min_vertical_offset: -150   # 管道生成时，空隙中心点Y轴相对于屏幕中心Y轴的最小偏移量。
  max_vertical_offset: 150    # 管道生成时，空隙中心点Y轴相对于屏幕中心Y轴的最大偏移量。
  initial_spawn_x_offset: 200 # 第一对管道生成时的X坐标额外偏移，相对于屏幕宽度。例如：屏幕宽度+此值。
  min_pipe_y_bottom: 150      # 管道底部（下管道）距离屏幕底部的最小距离，防止管道生成过低。
  max_pipe_y_top: 550         # 管道顶部（上管道）距离屏幕底部的最大距离，防止管道生成过高。

# ---------------------------------------------------
# 计分系统设置 (Scoring System Settings)
# ---------------------------------------------------
score_settings:
  score_per_pipe: 1           # 每成功穿过一对管道获得的积分数量。

# ---------------------------------------------------
# 碰撞检测设置 (Collision Detection Settings)
# ---------------------------------------------------
collision_settings:
  player_vs_pipe_action: "GameOver"   # 玩家与管道碰撞后的行为。
  player_vs_floor_action: "GameOver"  # 玩家与地面碰撞后的行为。
  player_vs_ceiling_action: "GameOver" # 玩家与天花板碰撞后的行为。
```

**致程序员和美术：**

*   **关于设计意图**:
    *   **难度平衡**: 初始数值旨在提供一个中等偏高的挑战体验。`game_speed`、`gravity`、`pipe_gap_height` 和 `horizontal_spacing` 是调整游戏难度的核心参数。`game_speed` 和 `gravity` 影响整体节奏感，而管道参数则直接影响玩家操作的精准度要求。
    *   **操作手感**: `jump_force_velocity` 和 `max_fall_speed` 共同决定了小鸟的跳跃高度和下落速度，是影响玩家操作手感的关键。建议在开发过程中多进行测试和微调，找到最舒适的平衡点。
    *   **随机性**: `min_vertical_offset` 和 `max_vertical_offset` 确保了管道位置的随机性，增加了游戏的可玩性和重复挑战的乐趣。
*   **美术资源**:
    *   小鸟、管道、背景图建议采用2D卡通风格，颜色鲜明，易于识别。
    *   小鸟需要一个简单的拍翅膀动画（2-3帧即可）。
    *   管道可以设计成上下两部分，中间留空。
    *   背景图需要设计成可循环滚动的形式。
*   **技术实现**:
    *   物理系统可以采用简单的基于速度和加速度的更新，或者利用现成的2D物理引擎（如Box2D、Unity的2D Physics）。
    *   碰撞检测可以采用简单的AABB碰撞检测，对于小鸟可以考虑更精细的圆形或胶囊体碰撞体。
    *   管道的生成和销毁需要考虑对象池机制，减少GC开销。
