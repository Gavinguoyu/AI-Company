### 1. 游戏概述 (Game Overview)

“1111”是一款基于经典贪吃蛇玩法的休闲益智游戏。玩家将控制一条在网格地图上移动的蛇，通过吃掉随机刷新的食物来增长身体。游戏目标是在避免撞到墙壁或自身的情况下，尽可能长时间地生存并获得高分。

---

### 2. 核心玩法 (Core Gameplay)

*   **移动与控制:** 玩家使用方向键（或触摸滑动）控制蛇头的移动方向，蛇身会跟随蛇头移动。
*   **食物与增长:** 蛇头触碰到食物时，食物消失，蛇身增长，玩家获得分数。
*   **失败条件:** 蛇头撞到地图边界（墙壁）或蛇身体的任何一部分时，游戏结束。
*   **高分挑战:** 随着蛇身增长，移动速度会逐渐加快，增加游戏难度，考验玩家的反应和策略。

---

### 3. 游戏元素 (Game Elements)

#### 3.1 蛇 (Snake)

*   **组成:** 由一个蛇头和若干个蛇身节组成，每个节占据地图上的一个网格单元。
*   **初始状态:** 蛇拥有初始长度（例如3节），并以预设方向开始移动。
*   **移动规则:**
    *   蛇以固定时间间隔自动向前移动一个网格单元。
    *   玩家可以随时改变蛇头的移动方向（上、下、左、右）。
    *   **重要限制:** 蛇不能立即180度反向移动。例如，如果蛇正在向右移动，玩家不能直接按左键使其立即反向。
*   **增长机制:** 每吃一个食物，蛇身增长一个节。新增长的节会添加到蛇尾。
*   **碰撞检测:**
    *   **与墙壁碰撞:** 蛇头触碰到地图边界（墙壁）时，游戏结束。
    *   **与自身碰撞:** 蛇头触碰到除自身以外的任何一个蛇身节时，游戏结束。

#### 3.2 食物 (Food)

*   **类型:** 基础食物（例如：一个苹果的图标）。
*   **生成规则:**
    *   每当蛇吃掉一个食物后，地图上会立即随机生成一个新的食物。
    *   食物必须生成在地图上一个空闲的网格单元内，即不能生成在蛇身或墙壁上。
*   **效果:**
    *   被蛇头吃掉后消失。
    *   使蛇身增长一个节。
    *   增加玩家分数。
    *   （可选，见难度设计）略微增加蛇的移动速度。

#### 3.3 地图 (Map/Grid)

*   **结构:** 一个矩形的网格区域，由等大的正方形单元格组成。
*   **边界:** 地图的四周是不可穿越的墙壁。
*   **尺寸:** 地图的宽度和高度是可配置的（例如 20x20 个单元格）。
*   **网格系统:** 所有游戏元素（蛇节、食物）都严格占据一个网格单元。

---

### 4. 游戏流程 (Game Flow)

*   **4.1 游戏开始 (Game Start):**
    *   游戏启动，直接进入游戏场景。
    *   初始化地图、蛇（初始位置、长度和方向）、食物（随机生成一个）。
    *   蛇开始自动向预设方向移动。
*   **4.2 游戏进行中 (In-Game):**
    *   玩家通过操作改变蛇的移动方向。
    *   游戏循环：蛇移动 -> 检测碰撞 -> 检测食物 -> 刷新画面。
    *   **吃到食物:** 蛇增长，分数增加，新的食物在随机空闲位置生成，（可选）蛇移动速度加快。
    *   **发生碰撞:** 游戏立即结束。
*   **4.3 游戏结束 (Game Over):**
    *   游戏画面暂停，蛇停止移动。
    *   显示“游戏结束”提示。
    *   显示本次游戏的最终得分。
    *   提供“重新开始”按钮，玩家点击后重置游戏并开始新一局。

---

### 5. 操作方式 (Controls)

*   **键盘 (PC/Web):**
    *   `↑` (Up Arrow): 蛇向上移动
    *   `↓` (Down Arrow): 蛇向下移动
    *   `←` (Left Arrow): 蛇向左移动
    *   `→` (Right Arrow): 蛇向右移动
    *   **注意:** 蛇不能进行180度反向操作（例如，蛇向右移动时不能直接按左）。

---

### 6. 得分系统 (Scoring System)

*   **基础得分:** 每吃一个食物，玩家获得固定分数（例如：10分）。
*   **总分计算:** 玩家的总分是所有吃掉食物得分的总和。
*   **目标:** 玩家的目标是尽可能获得更高的分数。

---

### 7. 难度设计 (Difficulty Design)

*   **初始速度:** 蛇的初始移动速度。这是游戏初期挑战性的主要来源。
*   **速度增长:**
    *   每吃一个食物，蛇的移动速度会略微加快。
    *   速度加快会使蛇的移动时间间隔缩短，从而要求玩家更快的反应速度和更精确的操作。
    *   设定速度上限，避免速度过快导致无法操作。
*   **地图大小:**
    *   更大的地图提供更多的操作空间，但可能需要更长时间才能找到食物。
    *   更小的地图则会限制操作空间，使游戏更具挑战性。
    *   当前版本默认为中等大小，未来可考虑作为可配置项。

---

### 8. 界面设计 (UI Design)

*   **游戏界面:**
    *   在屏幕的某个角落（例如左上角或右上角）持续显示当前玩家的得分。
    *   清晰地展示游戏区域、蛇、食物和墙壁。
*   **游戏结束界面:**
    *   在屏幕中央以醒目方式显示“游戏结束”文字。
    *   在“游戏结束”文字下方显示玩家的最终得分。
    *   提供一个明显的“重新开始”按钮。

---

### 9. 音效与音乐 (Audio)

*   **背景音乐:** 轻松、循环播放的背景音乐，以营造休闲的游戏氛围。
*   **音效:**
    *   **吃食物音效:** 清脆的音效，提示玩家成功吃到食物。
    *   **游戏结束音效:** 简洁而明确的音效，告知玩家游戏失败。

---

### 10. 技术可行性与优化建议

*   **网格系统:** 游戏基于简单的2D网格系统，技术实现难度较低。
*   **碰撞检测:** 采用基于网格坐标的碰撞检测，效率高且易于实现。
*   **性能优化:** 随着蛇身增长，需要注意渲染和碰撞检测的性能，但对于经典贪吃蛇来说，通常不是瓶颈。
*   **可配置性:** 大部分游戏参数已在配置表中列出，方便程序员进行调整和平衡。

## 游戏配置表 (Game Configuration - YAML)

以下是用于游戏核心参数的YAML配置文件。程序员可以直接导入和使用这些数据。

```yaml
# game_config.yaml

# 游戏通用设置
game_settings:
  title: "1111" # 游戏标题
  initial_game_state: "playing" # 游戏启动时直接进入游戏 ("playing") 或显示菜单 ("menu")

# 地图设置
map_settings:
  width_cells: 20 # 地图宽度，以网格单元数量计
  height_cells: 20 # 地图高度，以网格单元数量计
  cell_size_pixels: 30 # 每个网格单元在屏幕上渲染的像素大小 (供美术和渲染引擎参考)

# 蛇的设置
snake_settings:
  initial_length: 3 # 蛇的初始身体节数 (包括头部)
  initial_direction: "right" # 蛇的初始移动方向 (up, down, left, right)
  initial_head_position: [10, 10] # 蛇头初始的网格坐标 [x, y]，(0,0)为左上角
  growth_per_food: 1 # 每吃一个食物，蛇增长的节数
  collision_rules:
    wall_collision_ends_game: true # 蛇撞到墙壁是否游戏结束
    self_collision_ends_game: true # 蛇撞到自己身体是否游戏结束
  # 蛇不能立即180度反向的逻辑由程序实现，此处不作配置项，而是作为游戏规则在GDD中说明

# 食物设置
food_settings:
  type: "apple" # 食物类型 (目前只有一种，未来可扩展)
  score_per_food: 10 # 每吃一个食物获得的分数
  spawn_delay_ms: 0 # 食物生成延迟 (毫秒)，0表示立即生成

# 游戏难度与速度设置
difficulty_settings:
  initial_move_interval_ms: 300 # 蛇每移动一个单元格的时间间隔 (毫秒)。值越小，速度越快。
  speed_increase_per_food_percent: 0.02 # 每吃一个食物，蛇的移动速度提升的百分比 (例如0.02表示速度加快2%)
  min_move_interval_ms: 100 # 蛇移动的最小时间间隔 (毫秒)，达到此值后速度不再增加 (速度上限)

# UI文本配置 (未来可考虑多语言)
ui_texts:
  game_over_title: "游戏结束"
  restart_button: "重新开始"
  score_label: "分数: "
```
